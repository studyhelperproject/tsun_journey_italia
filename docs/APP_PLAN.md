# アプリ計画書：おぱのイタリアン脱出（仮）

## 1. アプリ概要
本作は、イタリアの食文化やマナーを題材にした「ゆる脱出ゲーム」です。プレイヤーは主人公「おぱ」がイタリアで禁忌を犯さないよう、様々なギミックを駆使して助けます。

- **タイトル**: おぱのイタリアン脱出（仮）
- **ジャンル**: カジュアル脱出パズルゲーム
- **ターゲット**: 全年齢（シュールな笑いを好む層）
- **プラットフォーム**: iOS (App Store), Android (Google Play)

## 2. キャラクター設定
- **おぱ (主人公)**:
  - 悪気はないが、ついついイタリアの食の禁忌（パスタをナイフで切る、午後にカプチーノを頼む等）を犯そうとしてしまう天然キャラクター。
- **つん (味方)**:
  - おぱの親友。おぱがピンチの時にヒントをくれたり、アイテムを差し出して助けてくれる。
- **だで (敵/お邪魔キャラ)**:
  - イタリアの伝統を重んじる厳しい存在。おぱが禁忌を犯すと、オペラを歌いながら絶望したり、激怒してゲームオーバー（失敗演出）を引き起こす。

## 3. 技術アーキテクチャ
本アプリは、クロスプラットフォーム開発を効率化するため、以下のスタックを使用します。

- **フレームワーク**: React Native
- **開発ツール**: Expo (Managed Workflow)
- **状態管理**: React Context API または Zustand
- **アニメーション**:
  - React Native Reanimated (滑らかな動き)
  - Lottie (複雑なキャラクターアニメーション)
- **音声**: expo-av (BGM/SE再生)
- **ストア配布**:
  - iOS: App Store
  - Android: Google Play Store
  - Expo Application Services (EAS) を使用してビルドおよび提出を実施。

## 4. UI/UX デザインとゲームメカニクス
画面は「メイン探索エリア」を中心に、直感的な操作ができるよう構成します。

### 画面レイアウト
- **上部：ヘッダー**:
  - ステージ番号（例：Stage 01）、ヒントボタン、メニューボタン。
- **中央：メインゲーム画面**:
  - 「おぱ」や怪しいオブジェクトが配置される。横スクロールが可能なステージもあり。
- **下部：フッター**:
  - アイテムスロット（最大5枠）。取得したアイテムをドラッグ＆ドロップで使用可能。

### 操作方法
- **タップ**: オブジェクトを調べる、アイテムを拾う、ギミックを作動させる。
- **ドラッグ＆ドロップ**: アイテムスロットからメイン画面の対象物へアイテムを運んで使用。

### 演出
- **成功演出**: 問題をスマートに解決し、「おぱ」が満足げな顔でクリア。
- **失敗演出**: 「だで」が登場し、大げさなアクション（絶望、怒り）とともにゲームオーバー。

## 5. ステージ構成案（全20ステージ）
「おぱ」が直面するイタリアの日常的な場面をベースに構成します。

### 【食のこだわり編】
- **Stage 01: カプチーノの門限**: 午後2時にカプチーノを注文しようとする「おぱ」を止める。
- **Stage 02: パスタの断髪式**: ナイフでパスタを短く切ろうとするのを阻止。
- **Stage 03: スプーンの赤ちゃん**: スプーンを使ってパスタを巻こうとするのを修正。
- **Stage 04: 禁断のケチャップ**: ナポリタンにケチャップを使おうとするのを防ぐ。
- **Stage 05: アルデンテの鉄則**: 茹ですぎた麺を「だで」に出さないようにする。

### 【生活・マナー編】
- **Stage 06: 教会のドレスコード**: 露出の多い服で教会に入ろうとする「おぱ」を着せ替える。
- **Stage 07: セルフィーの代償**: 噴水の縁で危険な自撮りをしようとするのを止める。
- **Stage 08: 挨拶の距離感**: 「つん」とのハグの回数や向きを正しく導く。
- **Stage 09: お会計のジェスチャー**: 大声で叫ばず、スマートに合図させる。
- **Stage 10: シエスタの休息**: 寝ている店主「だで」を起こさないように入店。

...（以降、ファッション・文化編、情熱・ジンクス編へと続く）

## 6. 開発ロードマップ
- **フェーズ 1: プロトタイプ開発**: コアとなる操作（タップ、アイテム使用）とステージ1〜3の実装。
- **フェーズ 2: コンテンツ制作**: 全20ステージのギミック詳細設計、グラフィック、サウンド制作。
- **フェーズ 3: ブラッシュアップ**: 失敗演出のバリエーション追加、全体の操作性改善、バグ修正。
- **フェーズ 4: リリース準備**: EAS Build を利用したアプリビルド、各ストアへの申請情報の作成。
- **フェーズ 5: リリース**: App Store および Google Play への公開。
